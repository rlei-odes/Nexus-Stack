# Secrets

The Secrets page shows every secret managed by your stack, pulled live from **Infisical**. It's read-only by design — secrets are generated by OpenTofu during Spin Up and written to Infisical as the single source of truth. Editing them anywhere else would be overwritten on the next deploy.

![Secrets page with folders](./assets/secrets-list.png)

## How it's organised

Secrets are grouped by folder, matching the structure in Infisical:

- `/` — global secrets shared across stacks (database root passwords, Cloudflare tunnel credentials)
- `/grafana`, `/keycloak`, `/infisical` … — per-stack folders (admin passwords, API tokens)
- `/integrations` — third-party credentials added via the [Integrations](./integrations) page

Click a folder row to expand it.

## Reading a secret

Values are hidden by default. Two ways to see them:

- **Hover** a row to reveal the value (auto-hides after a few seconds)
- **Click** a row to copy the value to the clipboard (you get a toast confirmation)

![Secret reveal on hover](./assets/secrets-reveal.png)

## Search

The filter box at the top matches on key name AND folder. Useful when you know the name of a credential but not which service it belongs to.

## Editing secrets

You can't edit from the Control Plane. Two options:

1. **Rotate via re-deploy** — generated secrets (database passwords, admin tokens) rotate on the next Spin Up. Tear down, Spin Up, done.
2. **Direct in Infisical** — for user-added secrets (e.g. a third-party API key), log in to Infisical at `https://infisical.<your-domain>` with the credentials shown on the Control Plane Secrets page.

## Why no edit?

Nexus-Stack treats OpenTofu as the source of truth for infrastructure secrets. If you edited a secret in the Control Plane that OpenTofu manages, the next Spin Up would overwrite your change — silent drift. The read-only view makes that impossible.
